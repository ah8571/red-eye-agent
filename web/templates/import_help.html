Import Format Help

Quick Format

One task per line. No context files.

Add input validation to the login form
Create a new API endpoint for user profiles
Update the README with installation instructions
Fix the broken unit test in test_auth.py

Markdown Format

Use Markdown headings for sections and bullet points for tasks. Indent lines with context: to specify relevant files.

## Safety
- Add input validation to the login form
  - context: web/auth.py, web/templates/login.html
- Implement rate limiting on the API
  - context: web/app.py, web/middleware.py

## Features
- Create a new API endpoint for user profiles
  - context: web/app.py, web/models.py
- Add a dashboard widget for recent activity
  - context: web/templates/dashboard.html, web/static/js/dashboard.js

## Documentation
- Update the README with installation instructions
  - context: README.md
- Write API documentation for the new endpoints
  - context: docs/api.md

YAML Format

Full checklist YAML structure with explicit fields.

tasks:
  - id: 1
    repo: my-repo
    description: Add input validation to the login form
    status: pending
    context_files:
      - web/auth.py
      - web/templates/login.html
  - id: 2
    repo: my-repo
    description: Create a new API endpoint for user profiles
    status: pending
    context_files:
      - web/app.py
      - web/models.py
  - id: 3
    repo: my-repo
    description: Update the README with installation instructions
    status: done
    context_files:
      - README.md

Notes

  • All formats require authentication to use.
  • Quick format tasks are assigned the selected repository and have empty context files.
  • Markdown format: tasks inherit the repository from the selection; context lines are optional.
  • YAML format: repository can be specified per task; the UI selection is ignored.
